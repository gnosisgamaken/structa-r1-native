# Structa v1.0 — Full-Screen Panel Spec

## Goal
Make the app feel like a real device panel: full-screen, structured, quiet, and premium.

## Non-negotiables
- Top system bar is always present
- Full-screen composition fills the panel
- No browser-app feeling
- Minimal text in the scene
- One primary interaction model
- Strong geometry and solid fills

## Layout rules
- System bar at the top with back, current state, time, battery
- Main field is a modular six-tile composition
- No visual clutter around the matrix
- Bottom log is tiny and restrained
- No extra buttons unless absolutely necessary

## Visual rules
- Use solid geometric motifs inside tiles
- Prefer circles, rings, squares, triangles, diamonds, and crosses
- Avoid thin iconography
- Keep label text out of the main scene when possible
- Use large shapes that can be read instantly on a small screen

## Motion rules
- The panel should react like a living object
- Tiles should pulse and shift subtly when activated
- Sequence should feel rhythmic and composed
- No particles, blur, or chaotic motion

## Input rules
- Tap a tile to select and activate
- Arrow keys cycle selection
- Enter / Space triggers the selected tile
- Back / Escape should be handled gracefully
- Swipes should cycle or trigger on mobile hardware

## Success criteria
- The screen feels native to the R1 hardware
- The scene reads as a designed panel, not a web dashboard
- The composition is stronger than the labels
- The aesthetic is close to a collectible geometric print
