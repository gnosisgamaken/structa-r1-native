# Structa v1.1 — Four-Node Panel Spec

## Goal
Make the first screen feel native, premium, and creative on Rabbit R1 by reducing visible complexity and strengthening the panel composition.

## Non-negotiables
- No in-app top system bar
- Full-screen panel composition only
- Four primary nodes visible on the first page
- Two deeper nodes hidden behind hold / reveal
- Minimal text in the scene
- No browser-app chrome feeling
- Strong, solid geometric motifs only

## Layout rules
- 2x2 primary grid fills the screen
- Primary nodes are large and readable
- Hidden nodes appear only on demand
- Bottom log is small and quiet
- No extra floating UI elements

## Visual rules
- Use the reference-panel language: solid fills, cropped geometry, large simple forms
- Prefer circle, square, triangle, diamond, ring, and cross motifs
- Avoid thin outline icons
- Labels should be tiny or absent in the main face
- Composition should feel like a designed object, not a dashboard

## Motion rules
- Motion should feel like the panel is switching state
- Use quick spring-like transitions
- No particles
- No blur
- No drifting or decorative motion
- Trigger should feel intentional and physical

## Input rules
- Tap to select/trigger
- Hold core or memory to reveal hidden nodes
- Arrow keys cycle the primary nodes
- Enter / Space triggers the current selection
- Swipe gestures should cycle or trigger on touch hardware

## Success criteria
- The screen reads as a premium instrument panel
- It feels native to the R1 and credible for creatives
- The composition is stronger than the labels
- The first page is calm, dense enough, but not cluttered
- Hidden depth exists without overloading the home state
